## Known Limits and Follow-up Work

- MVP is intentionally in-memory only; there is no persistence layer.
- VLM/LLM, robot, simulator, and network integrations are not implemented and are represented by deterministic local logic or explicit offline prediction imports only.
- Relation thresholds are configurable but simple; future work can add richer geometry, semantic priors, and sensor evidence models.
- `ObservationIngestor` accepts structured local observations only; `SceneObservation` JSON helpers read or write only explicit caller-supplied local paths. Real sensor, simulator, or model adapters remain intentionally out of scope.
- VLA planner returns structured skill anchors only; it does not perform motion planning or robot control.
- Oracle QA generation creates deterministic MVP datasets from existing graph
  state only; large-scale benchmark assembly and baseline prediction runners
  are follow-up work.
- QA evaluation scores explicit prediction JSONL artifacts only; delta report
  comparison reloads only the explicit candidate and baseline report paths
  recorded in the delta report; baseline runners consume explicit graph and QA
  files only, and offline external predictions must be imported from explicit
  local files. Real model integrations remain follow-up work behind offline or
  mocked boundaries.
- Graph evaluation uses exact object IDs and exact relation edges by default,
  with optional label-plus-center and room-aware label-plus-center matching for
  changed predicted object IDs plus confidence-weighted metrics and
  prediction-source confidence slices. Deterministic predicted graph and
  observation ingestion propagate source metadata for local analysis; real
  perception-backed predicted graph construction and confidence calibration
  remain follow-up work.
- Error attribution is deterministic and local; future external-model adapters
  should write offline prediction artifacts and consume report formats rather
  than calling model providers during default verification.
- Dashboard HTML is intentionally lightweight and static. Rich interactive UI,
  media previews and deeper visual per-action review remain follow-up work.
- Active EQA is a mock graph-step loop only; real navigation,
  simulator-backed next-best-view execution, Habitat, AI2-THOR task execution,
  and robot interaction remain follow-up work behind deterministic adapter
  boundaries.
